# 圆柱滚筒舞台改版设计

日期：2026-07-13

## 1. 背景与目标

当前版本使用固定工具栏、左侧缩略图和中央预览区，交互接近普通 PPT 浏览器；PowerPoint 页面固定导出为 `1920×1080`，对高分屏和细小文字不够清晰。

本次改版有两个目标：

1. 将页面导出质量提升到接近 4K，并严格保持原始幻灯片比例。
2. 把整个窗口改造成舞台：页面沿一个横向可旋转的圆柱滚筒排列，用户左右拖动寻找页面，点击中央页后进入完整单页舞台。

本次不加入摄像头、MediaPipe 或手势识别。鼠标滑动接口继续作为第二阶段手势映射的基础。

## 2. 方案选择

### 2.1 考察过的空间模型

- 主舞台 + 常驻底部页轨：容易理解，但仍有传统播放器感。
- 无限页面墙：自由度高，但页面缺少明确的当前焦点，单页演示操作复杂。
- 单页舞台与页面墙切换：兼顾沉浸和自由选页。

用户进一步明确了空间意象：页面不应是平面网格，而应贴在一个圆柱形滚筒上，通过左右移动滚筒寻找页面。

### 2.2 最终选择

采用“圆柱滚筒选页 → 单页完整舞台”的双状态结构：

- 滚筒负责浏览和选择。
- 中央页正对用户、尺寸最大，两侧页面沿圆柱曲面逐渐压缩、变暗并后退。
- 点击中央页进入完整舞台。
- `Esc` 或页面墙按钮返回滚筒，并保留当前页和滚筒位置。

## 3. 视觉方向

继续采用项目既定 Swiss 方向：

- 表面使用纯白或中性浅灰。
- 主色只使用 Yves Klein Blue `#002FA7`。
- 结构使用 1px 细分隔线和清晰网格。
- 页码作为显著的构图元素。
- 不使用渐变、装饰性纹理、悬浮卡片堆叠或传统固定侧栏。

页面内容始终是视觉主体。工具只保留在舞台边缘，并使用系统图标和工具提示。

## 4. 交互设计

### 4.1 滚筒状态

- 所有页面按顺序映射到圆柱角度。
- 左右拖动连续旋转滚筒。
- 拖动速度用于计算短距离惯性。
- 松手后吸附到最近页面。
- 第一页和最后一页不循环。
- 点击非中央页面时，先将其吸附到中央。
- 点击中央页面时，进入单页舞台。

### 4.2 单页舞台状态

- 当前页尽可能占满舞台，同时保持原始比例。
- 页面处于“适应窗口”状态时，左右拖动触发上一页或下一页。
- 页面放大后，拖动仅用于平移，不触发翻页。
- 双击在“适应窗口”和 `100%` 之间切换。
- 鼠标滚轮继续缩放，范围保持 `25%~400%`。
- 键盘快捷键继续支持 `PageUp/PageDown`、方向键、空格、`Home/End` 和 `F11`。
- `Esc` 优先退出全屏；非全屏时返回滚筒。

### 4.3 状态保持

- 滚筒与单页舞台共享同一个当前页索引。
- 返回滚筒时，当前页仍位于中央。
- 退出程序时继续保存当前页。
- 再次打开项目时首先进入滚筒，并定位到上次页面。

## 5. 清晰度与导出

### 5.1 导出尺寸

- 标准导出宽度为 `3840`。
- 高度根据 PowerPoint `PageSetup.SlideWidth/SlideHeight` 计算。
- 16:9 页面导出为 `3840×2160`。
- 4:3 页面导出为 `3840×2880`。
- 不强制写死高度，避免非 16:9 页面被拉伸。

计算结果需要取整且至少为 1 像素。项目元数据记录实际导出宽高。

### 5.2 缩略图

- 滚筒使用独立 JPEG，不直接使用 4K PNG。
- 缩略图最大宽度约为 `640px`，高度保持原比例。
- JPEG 使用高质量重采样和约 90 的质量参数。
- 中央页吸附完成后才加载对应 4K 原图。

### 5.3 Qt 显示

- 单页舞台使用原始 PNG。
- QGraphicsPixmapItem 使用高质量变换模式。
- 适应窗口只改变显示矩阵，不生成二次缩放文件。

## 6. 缓存升级

缓存 schema 从版本 1 升级为版本 2。

版本 2 增加或严格校验：

- 实际导出宽度和高度。
- 导出规格标识，例如 `powerpoint-4k-v1`。
- 缩略图规格。
- 页面数量、索引连续性以及每张图片可读性。

旧版本缓存自动判定失效。重新导出仍遵循临时目录写入、完整校验、原子替换正式缓存的流程。导出失败或取消不得覆盖已有有效缓存。

## 7. 模块设计

### `widgets/cylinder_carousel.py`

负责滚筒状态：

- 根据连续浮点偏移计算每页角度。
- 根据 `sin/cos` 计算横向位置、水平压缩、尺寸、透明度和层级。
- 处理拖动、速度估算、惯性和页码吸附。
- 发出“当前页变化”和“进入单页舞台”信号。

使用 QGraphicsScene 的 2D 透视模拟，不引入 OpenGL 或第三方 3D 依赖。

### `widgets/stage_workspace.py`

负责两种状态的切换：

- 组合 `CylinderCarousel` 和 `SlideViewer`。
- 保持共享的当前页索引。
- 处理进入、返回和状态恢复。
- 对主窗口暴露页面选择、翻页和视图模式接口。

### `widgets/slide_viewer.py`

在现有缩放和平移能力上增加：

- 适应窗口状态判断。
- 水平滑动距离和方向识别。
- 上一页、下一页信号。
- 双击适应窗口/100% 切换。
- 高质量图片变换。

### `app/main_window.py`

- 移除固定左侧缩略图和传统主工具栏布局。
- 使用 `StageWorkspace` 作为中央主体。
- 将打开、缩放、页面墙、全屏、导入进度和取消控制压缩到舞台边缘。
- 继续负责快捷键、导入线程、错误提示和最近项目恢复。

### 导出和数据层

- `ppt/powerpoint_exporter.py` 读取页面比例并计算 4K 尺寸。
- `models/slide_project.py` 升级 schema 默认值并记录渲染规格。
- `ppt/project_store.py` 校验 schema 2 和图片规格。

## 8. 数据流

1. 用户打开 PPT。
2. 导入器计算源文件缓存键并检查 schema 2 缓存。
3. 缓存失效时，PowerPoint COM 按真实比例导出 4K PNG。
4. Pillow 生成 640px 滚筒缩略图。
5. 项目加载后，滚筒先显示全部缩略图并定位当前页。
6. 滚筒吸附完成或进入单页舞台时加载当前 4K 原图。
7. 页面切换时预加载相邻原图，并通过有限容量缓存淘汰最久未使用页面。

## 9. 性能边界

- 100 页项目不一次性解码所有 4K PNG。
- 滚筒拖动过程只操作已加载的缩略图和图形变换。
- 4K 原图缓存仅保留当前页和少量相邻页。
- 缩略图列表在首次展示时可分批加载，避免窗口首帧长时间阻塞。
- 动画期间不重复从磁盘加载同一图片。

## 10. 异常处理

- 4K 导出失败时显示中文错误，不静默降级到 1080p。
- 源文件比例不可读取时报告导出错误，不猜测并拉伸页面。
- 页面图片缺失、损坏、尺寸不匹配或索引不连续时，缓存整体失效。
- 导出失败、取消或程序退出后关闭独立 PowerPoint COM 进程。
- 返回滚筒或切换页面时，如果原图暂未加载，继续显示清晰缩略图并在加载完成后替换。

## 11. 测试与验收

### 自动化测试

- 16:9 尺寸计算结果为 `3840×2160`。
- 4:3 尺寸计算结果为 `3840×2880`。
- schema 1 缓存判定失效。
- schema 2 的尺寸、页数、索引或图片异常时判定失效。
- 圆柱几何在中央、左右边界和拖动中间值时正确。
- 惯性结束后吸附到最近有效页，且不越界。
- 适应窗口拖动发出翻页信号；放大后拖动只平移。
- 进入和退出单页舞台后当前页不变。

### Windows 真机测试

- PowerPoint COM 分别创建并导出 16:9 和 4:3 样本。
- 校验 PNG 的实际像素尺寸和缩略图比例。
- 真实 Windows 平台截图检查滚筒与单页舞台没有重叠或文字截断。
- 鼠标拖动滚筒、吸附、进入舞台、滑动翻页和返回流程可连续完成。
- 100 页 PPT 不一次性加载全部原图。
- 测试结束后无残留 PowerPoint 或项目 Python 进程。

## 12. 完成标准

- 主界面不再呈现固定左侧缩略图 + 中央预览的普通 PPT 框架。
- 用户能在整窗圆柱滚筒上左右拖动并选择任意页面。
- 中央页能进入完整单页舞台并通过滑动翻页。
- 放大和平移不会误触发翻页。
- 16:9、4:3 页面均按原比例生成 4K 图片。
- 旧缓存自动升级，失败不破坏有效缓存。
- 自动化测试和 Windows PowerPoint 真机测试通过。

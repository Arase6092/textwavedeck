# 手势滚筒暗色渐变与舞台密度优化设计

日期：2026-07-14

## 1. 背景

当前手势滚筒使用三段蓝灰暗色渐变，中央页目标高度为舞台高度的 `58%`。在 `1440 × 900` 窗口中，中央 16:9 页面高度约为 `522px`，页面上下合计保留约 `378px` 空区；在 `1024 × 768` 窗口中也存在相似的视觉松散问题。背景中心色偏蓝灰，层次变化集中在单一冷色系，页面之外的大面积暗区显得平且空。

本设计只优化手势模式中的滚筒舞台和对应的模式重组转场。PPT 模式的满屏页面框架、键盘和鼠标交互不变；手势单页舞台的缩放、拖动、双击和滑动翻页不变。

## 2. 调研结论

Material 暗色主题建议底层使用深灰而非纯黑，并通过轻微提亮的表面色表达层级。高饱和颜色在暗背景上容易产生视觉振动，因此强调色应受限，内容表面应保持清晰的亮度层级。

参考：<https://m2.material.io/design/color/dark-theme.html>

对本项目的直接约束：

- 舞台以近黑深灰为底，不使用纯黑。
- 渐变通过亮度和低饱和色相变化建立纵深，不使用装饰光斑、渐变球或横向舞台线。
- PPT 页面保持原图颜色，背景不能与页面争夺视觉焦点。
- 中央页扩大后仍需露出两级侧页，让用户立即理解滚筒可横向浏览。
- 暗色层级主要依赖背景和页面亮度差，不继续叠加强烈蓝色辉光。

## 3. 已确认方向

用户在本地视觉对比页中选择了 B「石墨青绿」方案。

视觉目标：

- 气质：安静、专业、接近演示控制台和黑匣子剧场。
- 背景：近黑到低饱和石墨青绿的四段斜向渐变。
- 密度：中央页明显放大，但左右两级页面仍能形成可识别的边缘提示。
- 层级：中央页靠尺寸、清晰度和轻量边框成为焦点；侧页通过透视、透明度和亮度后退。
- 连续性：进入滚筒的 overlay 与真实滚筒使用同一套目标几何，转场完成后不跳位。

## 4. 颜色规格

舞台渐变使用四个固定停点：

| 位置 | 色值 | 角色 |
|---:|---|---|
| `0.00` | `#080A0B` | 顶部近黑底色 |
| `0.35` | `#111817` | 中段石墨过渡 |
| `0.64` | `#18221F` | 低饱和青绿提亮区 |
| `1.00` | `#090B0E` | 底部回落色 |

执行规则：

- QSS 背景和 `QPainter` 背景必须使用相同停点与色值。
- 渐变方向保持左上到右下，背景固定在舞台坐标，不随页面移动。
- `FOCUS_BLUE` 保持 `#3B6FFF`，仅用于焦点边框、进度和交互状态。
- 顶部、底部控制层继续使用独立暗色表面，不将青绿色扩散到所有控件。
- PPT 图片不得调色、模糊、降低饱和度或覆盖渐变。

## 5. 滚筒密度规格

### 5.1 共享舞台参数

| 参数 | 当前值 | 目标值 |
|---|---:|---:|
| 中央页目标高度 | 舞台高度 `58%` | 舞台高度 `67%` |
| 中央页最大宽度 | 无独立限制 | 舞台宽度 `86%` |
| 圆柱半径 | 舞台宽度 `47%`，最大 `620px` | 舞台宽度 `49%`，最大 `660px` |
| 中心纵向偏移 | `-8px` | `-4px` |
| 侧页纵深落差 | 最大 `110px` | 最大 `92px` |

中央页基础尺寸按以下顺序计算：

1. 先以舞台高度的 `67%` 计算目标高度。
2. 按当前 PPT 页面的宽高比计算目标宽度。
3. 若目标宽度超过舞台宽度的 `86%`，则以宽度上限反算高度。
4. 在 4:3、16:9 和其他比例下都保持原始宽高比。

该规则使宽屏桌面充分减少上下留白，同时避免窄窗口中中央页完全覆盖侧页。

### 5.2 侧页层级

- 继续同时展示中央页和左右两级侧页，共 5 页。
- `cylinder_pose()` 的基础缩放、横向压缩、透明度和 z 层级保持不变，避免改变用户已经熟悉的滚筒手感。
- 圆柱半径略增，让放大后的中央页与一级侧页之间仍保留可识别边缘。
- 纵深落差减小，使页面组更紧凑，不在中央页下方形成过多空区。
- 中央页蓝色阴影的 alpha 从 `100` 降到 `72`，焦点主要依靠页面尺寸和边框，不形成明显发光块。

## 6. 几何共享设计

在 `widgets/cylinder_geometry.py` 中增加纯几何结果类型和计算函数，用于统一真实滚筒与重组 overlay。

接口定义：

```python
@dataclass(frozen=True, slots=True)
class CarouselViewportGeometry:
    center_x: float
    center_y: float
    radius: float
    target_height: float
    max_page_width: float
    depth_drop: float


def carousel_viewport_geometry(width: float, height: float) -> CarouselViewportGeometry:
    ...


def fit_carousel_page(
    geometry: CarouselViewportGeometry,
    aspect_ratio: float,
) -> tuple[float, float]:
    ...
```

调用关系：

- `CylinderCarousel._layout_items()` 使用共享结果计算页面基础缩放、中心坐标、半径和纵深落差。
- `StageRecompositionOverlay._carousel_rect()` 使用相同结果计算转场目标矩形。
- 页面宽高比限制统一由 `fit_carousel_page()` 处理，两个调用方不得保留各自的尺寸公式或常量。
- 共享模块不依赖 Qt，便于直接测试不同分辨率和宽高比。

## 8. 响应式与边界行为

- 主窗口最小尺寸 `980 × 620` 下，中央页不得超出可交互舞台，一级侧页至少保留可见边缘。
- 16:9 页面在 `1440 × 900` 下中央页高度应接近舞台高度的 `67%`。
- 16:9 页面在 `1024 × 768` 下允许受 `86%` 宽度上限约束，不能为了追求固定高度而遮住全部侧页。
- 4:3 页面优先使用高度目标，保持上下边距并避免超出控制层覆盖区域。
- 第一页和最后一页因没有完整的双侧邻页，可以自然留出非对称空间，不用复制页面填满。
- 控制层显示或隐藏时不改变滚筒几何，避免自动隐藏导致页面跳动。

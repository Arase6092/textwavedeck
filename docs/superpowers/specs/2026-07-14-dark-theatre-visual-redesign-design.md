# 黑匣子剧场视觉改版设计

日期：2026-07-14

## 1. 背景

圆柱滚筒和单页舞台功能已经完成，但当前界面仍保留明显的传统桌面软件特征：顶部白色工具栏长期占据视觉焦点，状态条和按钮边框切割舞台，滚筒缺少稳定的空间参照，侧页与中央页的明暗层级不足。功能可用，但没有形成“整块屏幕是一座舞台”的体验。

本次改版只调整视觉、布局和动效，不改变 PPT 导入、4K 导出、缓存、翻页、缩放、全屏和键盘操作的业务行为。

## 2. 已确认方向

用户确认采用“黑匣子剧场”方向：

- 整个主窗口使用暗场，不只把内容区域改黑。
- PPT 页面保持原始颜色，成为舞台中的唯一主要亮面。
- 控制台默认自动隐藏，不长期占用舞台。
- 圆柱滚筒同时展示 5 页：中央页、两张第一层侧页和两张第二层侧页。
- 动效克制、可中断，并提供减少动态模式。

本设计经用户确认后，取代项目原有 UI 约定中的浅色 Swiss 表面规则。信息结构仍保持紧凑、理性、无装饰堆叠，但视觉锚点改为完整暗场。

## 3. 设计依据

### Material Design 3 Carousel

- 被压缩或部分露出的侧项能明确提示用户仍可继续滑动。
- 中心 Hero 适合使用吸附滚动。
- 滚动时项目可改变尺寸，但释放后应恢复稳定布局。
- 大屏可以显示更多可识别项目，中心项必须保持足够尺寸。

参考：<https://m3.material.io/components/carousel/guidelines>

### Apple Human Interface Guidelines: Motion

- 动效必须服务于理解，不能遮盖内容。
- 反馈动画应简短、精确，并允许用户立即打断。
- 大范围运动需要固定参照，避免让整个环境看起来都在移动。
- 外围区域的大幅运动和长期摆动可能造成不适。

参考：<https://developer.apple.com/design/human-interface-guidelines/motion>

### Fluent 2 Motion

- 惯性、速度和减速应符合物理预期。
- 重要元素获得主要动效，次要元素应同步、低调地退出。
- 大面积顶层切换适合快速淡入淡出，避免整屏滑行造成迷失。
- 动效时长要随元素尺寸和移动距离变化，但不能让用户等待。

参考：<https://fluent2.microsoft.design/motion>

### WCAG 2.2, Success Criterion 2.3.3

- 交互触发的非必要动画应允许关闭。
- 应读取操作系统的减少动态偏好，或提供应用内等效设置。
- 视差、页面翻转和大幅运动不能成为传递信息的唯一方式。

参考：<https://www.w3.org/WAI/WCAG22/Understanding/animation-from-interactions.html>

## 4. 视觉系统

### 4.1 颜色

| 角色 | 颜色 | 用途 |
|---|---|---|
| 舞台背景 | `#07080B` | 主窗口和全屏背景 |
| 控制表面 | `#111318` | 自动隐藏控制台、导入状态 |
| 悬停表面 | `#191D24` | 控件悬停与选中背景 |
| 结构分隔 | `#303641` | 1px 边线和固定参照线 |
| 主文字 | `#F4F6FA` | 文件名、当前状态、主要标签 |
| 次文字 | `#A8AFBC` | 辅助信息、未选中页码 |
| 禁用文字 | `#626A78` | 不可用控制项 |
| 焦点蓝 | `#3B6FFF` | 当前页、键盘焦点、进度和交互反馈 |
| 错误红 | `#FF5C68` | 导入失败与破坏性提示 |

不使用渐变、玻璃拟态、彩色光斑、持续发光或大面积蓝色表面。蓝色只用于焦点和状态反馈。

### 4.2 字体与图标

- 使用 Windows 可用的 `Segoe UI Variable`，回退到 `Segoe UI`。
- 页码使用等宽数字特性，避免数字变化造成横向跳动。
- 图标统一为白色线性图标，活动状态切换为焦点蓝。
- 图标显示尺寸统一为 18px，交互区域至少 40×40px。
- 不使用 Emoji、文字字符或混合粗细的图标替代品。

### 4.3 表面与深度

- 页面和控制台最大圆角为 4px。
- 中央页使用一条冷白边线和轻量蓝色投影，投影只强调页面离开背景的深度。
- 侧页不使用模糊；通过透视、亮度、饱和度和透明度形成层级。
- 舞台背景保持纯净，不加入纹理、网格、星点或装饰图形。

## 5. 主窗口布局

主窗口由舞台和覆盖层组成，不再使用固定工具栏与固定状态条占据布局空间。

### 5.1 顶部信息层

- 左侧：打开按钮和当前 PPT 文件名。
- 中央：固定页码，例如 `04 / 27`。
- 右侧：滚筒/舞台切换和全屏按钮。
- 高度 52px，覆盖在舞台上方，不改变舞台尺寸。

### 5.2 底部控制层

- 滚筒模式仅显示一条页面进度线。
- 单页舞台显示缩小、当前比例、放大和适应窗口。
- 导入时显示进度、状态和取消按钮。
- 高度 64px，覆盖在舞台下方，不推动 PPT 页面。

### 5.3 自动隐藏

- 鼠标停止操作 2 秒后，顶部和底部控制层淡出。
- 鼠标进入顶部 56px 或底部 72px 区域时显示对应控制层。
- 窗口刚打开、模式刚切换或导入状态变化时，控制层至少显示 1.5 秒。
- 导入进行中、错误提示出现或键盘焦点位于控制项时禁止自动隐藏。
- 键盘翻页、滚筒拖动和页面吸附本身不主动显示控制层。
- 全屏模式沿用相同规则，默认完全无界面。

## 6. 五页圆柱滚筒

### 6.1 可视层级

滚筒最多显示 5 页：

1. 中央页：全亮、完整颜色、最大尺寸，位于最高层级。
2. 第一层侧页：前后各一页，保留可辨识内容，降低亮度和饱和度，并产生明显圆柱透视。
3. 第二层侧页：再向外各一页，只保留约 20% 的视觉强度，接近舞台边缘轮廓。

屏幕较窄时不压缩页面文字到不可辨识尺寸。`1024×768` 仍显示 5 页，但第二层侧页可以只露出窄边；低于程序最小窗口尺寸不继续适配。

### 6.2 固定参照

- 背景、顶部页码和一条极弱的水平参照线保持静止。
- 页面只在固定参照区域内旋转。
- 参照线颜色使用结构分隔色，不穿过 PPT 正文形成明显遮挡。
- 不移动背景、不旋转整个世界、不使用持续视差。

### 6.3 焦点反馈

- 中央页使用冷白边线和焦点蓝投影。
- 侧页不显示页码标签，避免五个数字同时竞争注意力。
- 页面吸附完成后才更新顶部固定页码。
- 点击侧页先将其吸附到中央；点击已居中的页面进入单页舞台。

## 7. 单页舞台

- 页面在可用区域内尽可能放大并保持原始比例。
- 页面周围保留至少 32px 暗场安全边距，避免贴边产生普通图片查看器感。
- 适应窗口状态下水平拖动继续触发翻页。
- 页面放大后，拖动只平移图片，不触发翻页。
- 缩放控制只在控制层显示，不在页面上叠加常驻按钮。
- 返回滚筒时，当前页位于中央且页码不变化。

## 8. 动效规格

### 8.1 滚筒拖动与吸附

- 页面拖动实时跟随指针。
- 松手后加入短距离惯性，单次最多跨越 2 页。
- 吸附时长为 180–320ms，按移动距离调整。
- 使用自然减速曲线，不使用弹簧回弹或过冲。
- 下一次鼠标、滚轮或键盘输入立即终止当前动画。

### 8.2 滚筒与舞台切换

- 进入舞台时，中央页在原位置放大到单页尺寸。
- 第一层和第二层侧页在 180ms 内压暗并退出。
- 中央页变换时长约 260ms，始终保持相同内容和宽高比。
- 返回滚筒执行反向变换，并恢复 5 页层级。
- 不让整个窗口或背景横向滑出屏幕。

### 8.3 控制层

- 显示和隐藏只改变透明度，不改变舞台布局。
- 淡入淡出时长约 160ms。
- 控制层动画被用户交互打断时直接进入目标状态。

### 8.4 减少动态模式

- 关闭惯性和视差。
- 页面释放后直接定位到最近页，仅保留不超过 120ms 的淡入淡出。
- 滚筒与舞台切换改为快速淡变，不执行大幅缩放。
- 不使用持续呼吸、闪烁、循环光带或自动轮播。

## 9. 界面状态

### 9.1 未打开项目

- 整体保持暗场。
- 中央显示大号 `00`、文件图标和“打开 PPT”主操作。
- 不显示功能说明、快捷键列表或装饰性宣传文案。

### 9.2 导入中

- 控制层强制显示。
- 底部进度线使用焦点蓝，状态文本使用主文字。
- 取消按钮保持可见且不被自动隐藏。
- 舞台不显示伪造页面或占位缩略图。

### 9.3 导入成功

- 进入 5 页滚筒并定位到上次页面。
- 短暂显示“缓存命中”或“4K 页面已导出”，随后恢复页码状态。

### 9.4 导入失败

- 保持暗场背景，不切换到白色错误页。
- 使用紧凑错误弹窗显示中文原因和关闭操作。
- 焦点回到打开按钮，允许立即选择其他文件。

### 9.5 全屏

- 默认隐藏所有控制和系统化状态文本。
- 仅在边缘唤出控制层。
- `Esc` 仍先退出全屏，再按一次返回滚筒。

## 10. 模块边界

### `app/theme.py`

集中定义颜色、字体、间距、图标尺寸、控制层尺寸和动效时长。组件通过语义名称使用令牌，避免散落硬编码颜色。

### `widgets/stage_chrome.py`

负责顶部信息层、底部控制层、边缘唤出、超时淡出、导入锁定和键盘焦点保持。对主窗口暴露状态更新接口，不负责 PPT 导入业务。

### `widgets/cylinder_geometry.py`

定义中央、第一层侧页和第二层侧页的横向位置、比例、水平压缩、透明度和层级，并提供减少动态时的简化姿态。

### `widgets/cylinder_carousel.py`

应用暗场绘制与 5 页层级，限制惯性最多跨 2 页，处理吸附和动画中断。页面选择信号保持现有接口。

### `widgets/stage_workspace.py`

负责滚筒与单页舞台的过渡、当前页同步和减少动态策略。业务层仍只接收页面选择和模式变化信号。

### `app/main_window.py`

移除现有固定白色工具栏和状态条，组合舞台与覆盖层；继续负责导入线程、错误提示、最近项目恢复、全屏和快捷键。

## 11. 数据与错误处理

- 本次不修改缓存 schema、4K 导出规格或 PowerPoint COM 流程。
- 控制层只观察导入状态，不直接操作工作线程。
- 自动隐藏计时器在窗口关闭、导入结束和控件销毁时停止。
- 图片缺失或损坏继续沿用现有中文错误处理，不用暗场效果掩盖错误。
- 减少动态偏好读取失败时使用普通克制动效，不阻止程序启动。

## 12. 测试与验收

### 自动化测试

- 圆柱几何在中心、第一侧层、第二侧层和不可见范围返回正确姿态。
- 惯性目标最多跨越 2 页，且不超出首尾页。
- 动画被新输入打断后，当前页和浮点偏移保持一致。
- 自动隐藏在 2 秒后触发，边缘进入后恢复显示。
- 导入中、错误状态和键盘焦点状态禁止自动隐藏。
- 减少动态模式不启动惯性和大幅缩放动画。
- 进入和退出单页舞台后当前页不变化。

### 视觉测试

- 在 `1440×900` 和 `1024×768` 捕获未打开、5 页滚筒、控制层显示、控制层隐藏、单页舞台、导入中和错误状态截图。
- 使用真实 PPT 页面补充截图，确认白底、深色和高饱和页面都能在暗场中正确呈现。
- 检查页面不重叠、按钮不截断、页码不跳动、第二层侧页不抢占焦点。
- 检查顶部和底部控制层不会遮挡 PPT 的关键信息区域。

### Windows 真机回归

- 鼠标拖动、滚轮翻页、点击侧页、进入舞台和返回滚筒连续可用。
- 自动隐藏、边缘唤出、全屏和 `Esc` 两级退出行为正确。
- 键盘翻页、缩放、适应窗口和最近项目恢复不回归。
- PowerPoint 16:9 与 4:3 导入、4K 尺寸、缓存命中和 COM 清理继续通过。
- 测试结束后无残留 PowerPoint 或项目 Python 进程。

## 13. 不在本次范围

- 摄像头、MediaPipe 和手势识别。
- PPT 动画、视频或音频播放。
- 内容编辑、批注和激光笔。
- 主题切换或浅色模式。
- 自定义图标库、OpenGL、Qt Quick 3D 或新的第三方依赖。

## 14. 完成标准

- 主窗口完全呈现黑匣子暗场，不再像带固定工具栏的普通 PPT 浏览器。
- 5 页滚筒具有明确中央焦点、两级侧页和稳定参照。
- 控制层默认自动隐藏，并能通过边缘、导入状态和键盘焦点可靠恢复。
- 滚筒动效克制、可打断，惯性最多跨 2 页。
- 减少动态模式可用，信息不依赖动画传达。
- 空状态、导入、成功、失败、单页舞台和全屏状态保持统一视觉语言。
- 自动化测试、视觉测试和 Windows PowerPoint 真机回归全部通过。

# 项目执行规则

## 开始前与流程分级

- 开始代码、测试或打包前，先读本文件和 `第一阶段执行方案.md`，确认阶段范围；再按任务涉及模块读取相关代码、测试及错误记录。
- 按需求复杂度、影响范围、回滚难度和不确定性选择流程。小功能、局部样式、明确小修复、文案/配置或单文件低风险改动默认走轻量流程：定位影响、最小修改、定向验证、报告结果。
- 轻量任务不强制执行完整 Superpowers 链路；brainstorming、writing-plans、TDD、subagent、code-review 和 finishing 等技能，仅在能降低当前风险时按需选用。
- 跨模块、架构、共享状态/数据格式、并发/性能/安全、大范围 UI、需求不清、难回滚或高影响任务，升级到相应 Superpowers 计划、测试、评审和验证流程。用户明确指定技能或完整链路时遵从。
- 执行中发现范围扩大时立即升级流程；所有任务都必须完成与风险相称的验证。

## 第一阶段边界

- 仅负责 Windows 桌面 PPT 图片浏览：导入 `.ppt/.pptx`、PowerPoint COM 导出 PNG、缓存、缩略图、预览、缩放、拖动、翻页、全屏和键盘操作。
- 不接入摄像头、MediaPipe、手势、动画/音视频、内容编辑、批注或激光笔。
- 仅在本机测试；不上传 PPT，也不把 PPT 内容写入日志。
- 导出先写临时目录，成功校验后再替换缓存；取消或异常不得覆盖有效缓存。
- 用户状态、异常和日志尽量用中文；函数及特殊处理必须有简短中文注释。
- 程序不得自动安装 PowerPoint、Python 或第三方包；缺少前提时给出明确提示。

## UI 硬约束

- 黑匣子剧场风格：主窗口使用 `#080A0B`、`#111817`、`#18221F`、`#090B0E` 石墨青绿四段渐变；PPT 保持原色；焦点色为 `#3B6FFF`。
- 滚筒同时展示中央页和两级侧页，共 5 页；背景不随页面移动，不绘制横穿舞台的水平参照线。
- 顶部和底部控制层默认 2 秒后自动隐藏，通过屏幕边缘、导入状态或键盘焦点恢复。
- 动效简短且可中断；惯性最多跨 2 页，并提供关闭惯性和大幅过渡的减少动态模式。
- 工具按钮使用一致的单色图标和工具提示；标准操作保留标准中文名称，不使用装饰性占位文案。
- 无项目、导入中、导入成功、导入失败和全屏状态都必须可理解。

## 已知边界路由

完整历史见 `docs/agent-error-log.md`，仅按任务读取相关章节：

- 启动、依赖或打包：读取“环境、依赖与启动”。
- PPT 导入、缓存或 COM：读取“PowerPoint 与导入”。
- 工作树、补丁或计划文件：读取“工作区与工具”。
- UI 状态、Qt 或界面测试：读取“UI 与 Qt 测试”。

发现可复用错误或边界时，按 `日期 | 文件/命令 | 现象 | 根因 | 修复/预防` 追加到对应章节，不把历史明细重新堆回本文件。

## 交付前检查

- 运行与改动风险匹配的核心自动化测试并记录结果。
- 检查 `AGENTS.md`、README、依赖文件和启动 `.cmd` 是否存在。
- 无 PowerPoint 环境启动时仍显示中文可操作提示，不得直接崩溃。

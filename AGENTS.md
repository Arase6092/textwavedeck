# 项目执行规则

## 每次执行前回顾

开始任何代码、测试或打包操作前，先阅读本文件和 `第一阶段执行方案.md`，确认当前工作仍在对应阶段范围内。完成后把新发现的可复用错误和边界补充到“错误记录”。

## 项目边界

- 第一阶段只负责 Windows 桌面 PPT 图片浏览：导入 `.ppt/.pptx`、PowerPoint COM 导出 PNG、缓存、缩略图、预览、缩放、拖动、翻页、全屏和键盘操作。
- 第一阶段不接入摄像头、MediaPipe、手势识别、动画/音视频播放、内容编辑、批注或激光笔。
- 仅在本机测试环境操作；不上传 PPT 内容，不记录 PPT 内容到日志。
- 导出必须先写临时目录，成功校验后再替换缓存；取消或异常不得覆盖已有有效缓存。
- 用户可见状态、异常和日志尽量使用中文；函数和特殊处理必须有简短中文注释。
- 不把 PowerPoint、Python 或第三方包安装动作写入程序自动执行；缺少运行前提时给出明确提示。

## UI 约定

- 采用黑匣子剧场视觉方向：整个主窗口使用 `#07080B` 暗场，PPT 页面保持原色，焦点色使用 `#3B6FFF`。
- 滚筒同时展示中央页和两级侧页共 5 页；背景与固定参照不随页面移动。
- 顶部和底部控制层默认在 2 秒后自动隐藏，通过屏幕边缘、导入状态或键盘焦点恢复显示。
- 动效必须简短、可中断；惯性最多跨 2 页，并提供关闭惯性和大幅过渡的减少动态模式。
- 工具按钮使用一致的单色图标和工具提示；标准操作保留标准中文名称，不使用装饰性占位文案。
- 界面需要在无项目、导入中、导入成功、导入失败和全屏状态下都可理解。

## 错误记录

格式：`日期 | 文件/命令 | 现象 | 根因 | 修复/预防`

- 2026-07-13 | 初始执行 | 项目目录没有现成代码和 Git 仓库 | 目录仅包含执行方案 | 首次执行先创建骨架并初始化本地 Git；不覆盖已有方案文档。
- 2026-07-13 | `python main.py` | 当前解释器未安装 PySide6 | 工作区只提供基础 Python/Pillow，尚未创建项目虚拟环境 | 入口捕获缺失依赖并输出中文安装命令；交付时使用 `.venv` 安装 `requirements.txt`。
- 2026-07-13 | `启动手势控制PPT.cmd` | 双击后命令被截断并出现中文乱码 | UTF-8 无 BOM 文件使用 LF 换行，`cmd.exe` 未按预期解析批处理边界 | 启动器控制文本保持 ASCII，并强制保存为 ASCII + CRLF；自动验证时用 `GESTURE_PPT_NO_PAUSE=1` 禁止暂停。
- 2026-07-13 | `tests/integration_powerpoint_smoke.py` | 直接运行时找不到 `ppt` 包 | Python 将脚本所在的 `tests` 目录作为首个搜索路径 | 手动集成脚本在导入应用包前显式加入项目根目录。
- 2026-07-13 | `PowerPointExporter.export` | PowerPoint 2016 报错“不允许隐藏应用程序窗口” | COM 不允许设置 `Application.Visible=False` | 保留独立 `DispatchEx` 实例默认可见性，仅通过 `Presentations.Open(..., WithWindow=False)` 隐藏演示文稿窗口。
- 2026-07-13 | `pip install -r requirements.txt` | 默认 PyPI 下载 PySide6 长时间无输出 | 当前网络对 `files.pythonhosted.org` 大文件传输不稳定 | 改用 `https://pypi.tuna.tsinghua.edu.cn/simple` 镜像，本次约 28 秒完成全部依赖安装。
- 2026-07-13 | PowerPoint COM 清理 | `Presentation.Close()` 偶发失败后残留 PowerPoint 进程和锁文件 | `Close()` 与 `Quit()` 位于同一 `try`，前者抛错会跳过后者 | 每个 COM 清理步骤必须独立捕获异常，确保 `Application.Quit()` 和 `CoUninitialize()` 始终继续执行。
- 2026-07-14 | 暗场视觉实施计划 | 验证命令引用不存在的 `tests/test_slide_navigation.py` | 计划沿用了第一阶段文档中的旧测试文件名 | 执行计划前先用 `rg --files tests` 核对实际测试文件；本项目导航与舞台交互回归使用 `tests/test_stage_interactions.py`。
- 2026-07-14 | 功能工作树回顾阶段方案 | 工作树中找不到 `第一阶段执行方案.md` | 该阶段方案只保留在项目根目录，没有被 Git 跟踪到功能工作树 | 从项目根目录读取阶段方案，从功能工作树读取实施计划和代码，不在工作树中硬编码查找未跟踪文档。

## 交付前检查

- 运行核心自动化测试并记录结果。
- 检查 `AGENTS.md`、README、依赖文件和启动 `.cmd` 是否存在。
- 在没有 PowerPoint 的环境中启动时，程序仍应显示中文可操作提示，而不是直接崩溃。

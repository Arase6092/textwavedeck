# 手势控制 PPT：第一阶段执行方案

## 1. 阶段目标

第一阶段先完成一个可独立使用的 Windows 桌面 PPT 图片浏览器：用户可以导入 `.ppt` 或 `.pptx`，程序将每页导出为 PNG，并通过缩略图、预览区、缩放、拖动、全屏和键盘完成浏览。

本阶段暂不接入摄像头、MediaPipe 和手势识别，但页面浏览能力统一封装为命令接口，第二阶段可直接将手势映射到这些命令。

## 2. 交付范围

### 本阶段完成

- 导入 `.ppt`、`.pptx`
- 使用 Microsoft PowerPoint 导出逐页 PNG
- 导出进度、取消和中文错误提示
- PPT 转换结果缓存
- 左侧缩略图列表
- 中间当前页预览
- 点击缩略图切换页面
- 上一页、下一页
- 鼠标滚轮缩放
- 按住鼠标拖动画面
- 适应窗口、原始比例、重置缩放
- 全屏浏览
- 键盘翻页
- 最近一次打开项目恢复
- 中文日志和异常提示

### 本阶段不完成

- 摄像头采集
- 手部关键点检测
- 手势识别
- PPT 动画、视频和音频播放
- PPT 内容编辑
- 演讲批注和激光笔

## 3. 运行前提

- Windows 10/11 64 位
- Python 3.11
- Microsoft PowerPoint 2016 或更高版本
- PySide6
- pywin32
- Pillow
- pytest

首版仅使用 PowerPoint COM 作为转换引擎，暂不维护 LibreOffice 备用路径。

## 6. 缓存设计

缓存目录：

```text
%LOCALAPPDATA%\GesturePPT\projects\
```

每个 PPT 使用“绝对路径、文件大小、最后修改时间”的 SHA-256 摘要作为缓存键：

```text
projects/
└── <sha256>/
    ├── project.json
    ├── slides/
    │   ├── slide_001.png
    │   └── slide_002.png
    └── thumbnails/
        ├── slide_001.jpg
        └── slide_002.jpg
```

`project.json` 保存以下信息：

```json
{
  "schema_version": 1,
  "source_path": "D:\\演示文件\\产品介绍.pptx",
  "source_size": 2048000,
  "source_modified_at": 1783900800,
  "slide_count": 20,
  "current_slide": 0,
  "export_width": 1920,
  "export_height": 1080,
  "created_at": "2026-07-13T10:30:00+08:00"
}
```

仅当源文件未变化、索引可解析、页数匹配且所有图片可读时命中缓存。否则重新导出。

## 7. PPT 导出流程

导出尺寸建议为 `1920 × 1080`，以保证字体、公式、图表和 SmartArt 的清晰度。

1. 在工作线程中初始化 COM。
2. 检查是否能够创建 PowerPoint COM 实例。
3. 以只读、隐藏窗口方式打开 PPT。
4. 在临时目录中执行 `Presentation.Export()`。
5. 按实际页序读取导出结果。
6. 统一命名为 `slide_001.png` 格式。
7. 使用 Pillow 生成 `240 × 135` 左右的缩略图。
8. 校验页面数量和图片可读性。
9. 导出成功后原子替换正式缓存目录。
10. 在 `finally` 中关闭演示文稿和 PowerPoint 进程。

必须先写临时目录，成功后再替换正式缓存，避免中途取消或 PowerPoint 崩溃产生残缺缓存。

## 8. 界面布局与交互

```text
┌──────────────────────────────────────────────────────────┐
│ 打开  上一页  下一页  缩小  100%  放大  适应  全屏       │
├───────────────┬──────────────────────────────────────────┤
│ 第 1 页缩略图  │                                          │
│ 第 2 页缩略图  │              当前幻灯片                  │
│ 第 3 页缩略图  │                                          │
│ ...           │                                          │
├───────────────┴──────────────────────────────────────────┤
│ 第 1 / 20 页 | 100% | 已加载                             │
└──────────────────────────────────────────────────────────┘
```

交互规则：

- 单击缩略图：切换到指定页面
- 鼠标滚轮或 `Ctrl + 鼠标滚轮`：缩放
- 鼠标左键拖动：移动已放大的图片
- 双击预览区：在“适应窗口”和 `100%` 间切换
- `PageDown`、`Right`、`Space`：下一页
- `PageUp`、`Left`：上一页
- `Home`、`End`：第一页、最后一页
- `F11`：进入或退出全屏
- `Esc`：退出全屏
- `Ctrl+O`：导入 PPT

到达第一页或最后一页时不循环跳转。

## 9. 性能策略

- 缩略图使用独立 JPEG，不直接缩放原图列表。
- 当前页、前一页和后一页优先加载原图。
- 页面切换时预加载相邻页面。
- PPT 转换、缩略图生成和大图解码不在主线程执行。
- 缩略图列表采用延迟加载。
- 图片缓存设置容量上限，淘汰最久未使用页面。
- 导入期间显示进度，保持窗口可响应。

## 10. 异常处理与日志

需要覆盖：PowerPoint 未安装、文件不存在、密码保护、文件损坏、缓存目录无权限、磁盘空间不足、导出图片缺失、用户取消和 COM 异常。

日志路径：

```text
%LOCALAPPDATA%\GesturePPT\logs\gesture-ppt.log
```

日志记录导入路径、缓存命中情况、PowerPoint 生命周期、导出耗时、页面数量、图片校验结果和异常堆栈，不记录 PPT 内容。

## 12. 测试方案

### 自动化测试

- 缓存键在文件变化后发生变化
- `project.json` 正确保存和恢复
- 缺页、损坏和版本不匹配缓存判定为失效
- 页码边界和直接选页行为正确
- 缩放比例限制在 `25%~400%`
- 导入失败不覆盖已有有效缓存
- 页面排序不受中文名称和系统区域设置影响

### Windows 集成测试

- 1 页空白 PPT
- 16:9 和 4:3 PPT
- 50 页图片型 PPT
- 100 页以上 PPT
- 含中文字体、公式、图表和 SmartArt 的 PPT
- 损坏文件和受密码保护文件
- 包含中文、空格和特殊字符的路径

## 13. 完成标准

- `.ppt` 和 `.pptx` 均可导入
- 页面数量、顺序与原 PPT 一致
- 中文、公式、图表和 SmartArt 显示正常
- 未修改的 PPT 再次打开可命中缓存
- 导出期间窗口保持响应
- 取消导入不会留下无效缓存
- 缩略图、上一页和下一页行为正确
- 缩放、拖动和适应窗口稳定
- 全屏和快捷键可用
- 100 页 PPT 不一次性解码全部原图
- PowerPoint 异常后程序能清理并给出中文错误
- 核心自动化测试通过
- 至少完成一次 Windows 真机集成测试
